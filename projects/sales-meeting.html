영업 미팅 현황 관리

배경

영업 미팅과 출장을 카톡과 사내 화이트보드에 이중으로 보고하고, 월말이면 영업담당자가 한 달치 기록을 확인해 일비를 손으로 취합했습니다.

문제 해결

회사 구글 계정으로 바로 쓰는 모바일 웹앱으로 옮겨, 현장에서 등록하면 현황·팀 통계·일비 정산이 자동으로 정리되도록 만들었습니다. 별도 서버 없이 구글 시트를 데이터 저장소로 사용합니다.

결과

현황 조회 3~5분 → 3초 · 월말 일비 취합 30분 → 즉시

07 · 기술 선택

왜 웹 배포가 아니라 Google Apps Script인가

IT팀도 서버 예산도 없는 1인 환경. ‘새 시스템 도입’이 아니라 ‘이미 있는 구글 계정 활용’이 유일하게 가능한 선택이었습니다.

09 · 보안 ①

데이터 보안 — 원본을 건드리지 않는 단방향 읽기

  • 동기화 스크립트는 사내 공유 폴더의 엑셀을 읽기 전용(openpyxl)으로 열어 내용만 추출 — 원본 파일은 수정·잠금하지 않음
  • 90초마다 파일 수정시각(mtime)만 비교 → 변경이 있을 때만 다시 읽음. 변경 없으면 아무 동작도 안 함
  • 추출 데이터는 Google Sheets에 덮어쓰기로 반영, 웹앱은 이 Sheets만 조회 — 외부 사용자는 NAS·원본에 직접 접근하지 않음
  • Google API 권한은 spreadsheets + drive.readonly로 최소화 — 드라이브 파일 생성·삭제 권한 없음
외부로 나가는 건
가공된 조회 데이터뿐.
원본은 사내망 안에.

접근 제어 — 등록된 이메일만, 모든 행위 로그

  • 일반/관리자 권한 분리 (타인 미팅 수정·삭제, 사용자 관리는 관리자만)
  • 본인 등록 건만 수정·삭제 (checkMeetingAccess)
  • 모든 등록·수정·삭제·복귀에 writeLog 감사 로그
